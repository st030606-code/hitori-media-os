# Campaign Overview: building-hitori-media-os

Status: ready-for-human-review

## Target Reader

- ひとりで発信・商品化・メディア運営を進めたい個人
- note / X / Substack / YouTube / Podcastを横断して発信したい人
- 同じテーマを媒体ごとに何度も書き直している発信者
- API自動化より先に、発信の型を整えたい個人クリエイター
- AIワークフローを自分の運営に取り入れたい開発者・編集者

## Positioning

発信を頑張るより、発信が回る仕組みを作るための実験ログ。完成版ツールの宣伝ではなく、Hitori Media OS を building-in-public で組み立てる過程と判断を共有する publication。

## Core Thesis

発信を頑張るより、発信が回る仕組みを作る。AIで記事を量産する前に、AIが使える知識DBと制作ワークフローを設計したほうが、ひとりメディア運営は長く続けやすい。

## Why This Campaign Matters

- ひとりメディア運営は「発信量」より「制作の型」で詰まりやすい、という認識を共有する。
- AI を「記事の量産機」ではなく「素材を構造化して再利用する装置」として位置付けるパターンを提示する。
- 完成品の発表ではなく、開発過程・迷い・手動運用の現実を building-in-public で見せる。
- 読者が自分の運営に取り入れやすい「最初の1ステップ」を1つだけ持ち帰れる設計にする。

## How This Supports Hitori Media OS

- 1つの Content Idea から複数媒体への展開フローを、自分自身の発信で証明する。
- Publish Package Builder / placeholder 検出 / `--replace-placeholder-package` / dry-run の運用を、実際のキャンペーンで通す。
- Sanity 戦略レイヤー（publicationStrategy / postPlan / notesPlan / growthAction）を、最初のキャンペーンで埋めて UI 検証を兼ねる。
- 開発過程そのものが、次のドキュメント・素材・教材の元レコードになる。

## How This Supports Substack Reader-List Growth

- Substack を「単なる投稿先」ではなく、publication / email / Notes / archive / subscriber asset として扱う。
- Welcome Email、About Page、subscriber CTA、Notes follow-up plan を strategy module と整合させる。
- discovery（X / Threads）→ deep trust（Substack Post + email）→ Japanese archive（note）→ deeper trust（YouTube / Podcast）の流れを設計する。
- paid化は急がない。free public work で信頼を作ってから初めて検討する。

## What Should Not Be Overclaimed

- Hitori Media OS は **完成版ではない**。手動運用が中心。
- 自動投稿、API 投稿、Sanity direct write、画像 / 動画生成 API はまだ入れていない。
- AI clone voice / AI-generated avatar は本人承認まで保留。
- subscribers の数や成果はまだ実績がない。先に「考え方」と「制作フロー」を共有する段階。
- 完成版ツールを売る前提ではなく、開発過程そのものが価値という立場で書く。

## Voice / Content / Format

- Voice: 落ち着いた、抑制的、誠実、building-in-publicの判断と迷いをそのまま見せる。
- Content: Hitori Media OS の開発ログを中心に、AIワークフロー / Substack戦略 / ひとりメディア運営の制作判断を扱う。
- Format: 週次 build log を主軸に、思想記事 / 実験結果 / 編集後記 を混ぜる。冒頭フックは「自分の現在地」から入る。

## Safety

- 完成度を盛らない / 数字を盛らない / paid offerを急がない
- 自動投稿しない / API 不使用 / Sanity direct write しない
- private/ 配下のPDFや有料教材本文を本文・スクショに含めない

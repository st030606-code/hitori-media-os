# Hitori Media OS v0.2 Demo Campaign: ai-blog-db

このpackageは、1つのContent Ideaから横断media campaignを作れることを確認するためのhuman-review draftです。

## Campaign Goal

「記事を増やすより、AIが使えるDBを作る」というcore messageを、note、Substack、X、Threads、Instagram、YouTube、Shorts、Podcastへ展開します。

## Target Reader

- ひとりメディア運営者
- note / X / YouTube / Podcastなどを横断して発信したい人
- AIを使いたいが、API自動化より先に発信の型を整えたい人
- 同じテーマを媒体ごとに何度も書き直している人

## Core Message

これからのブログは、記事数を増やすだけではなく、AIが使える構造化DBを持っているかが重要になる。

## Platform Roles

- note: long-form Japanese article / search archive
- Substack Post: trust / email / subscriber asset
- Substack Notes: interaction / discovery
- X: sharp hook / discovery
- Threads: conversational discovery
- Instagram: carousel / visual explanation
- YouTube long-form: authority / deeper trust
- Shorts: short hook / reach
- Podcast: reflective trust / audio relationship

## Production Modes

- text: ai-assisted
- visual: manual-chatgpt / existing local image
- YouTube: human-shot or hybrid
- Shorts: hybrid
- Podcast: human-recorded first, ai-clone future
- publishing: manual

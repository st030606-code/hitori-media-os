# Repurpose Map

## note article -> Substack Post

Convert the article from論考 into a personal build log. Keep the core thesis, but add what was built and what will be tested next.

## note article -> X thread

Extract the sharpest claims. Keep each post focused on one message.

## note article -> Threads discussion

Make it softer and more conversational. Invite replies.

## note article -> Instagram carousel

Turn the main contrast into visual slides:

1. 記事の山
2. AIが使えるDB
3. 1つの知識レコード
4. 複数媒体へ展開
5. まず手動で検証

## note article -> YouTube script

Use the article as the argument structure. Add spoken transitions, examples, and screen insert cues.

## YouTube script -> Shorts

Extract:

- "記事を増やすより、AIが使えるDBを作る"
- "完成記事ではなく、再利用できる知識レコード"
- "API化は最後でいい"

## YouTube / Podcast notes -> Substack Notes

Use recording insights as Notes:

- what felt clearer after recording
- what readers might disagree with
- what will be tested next

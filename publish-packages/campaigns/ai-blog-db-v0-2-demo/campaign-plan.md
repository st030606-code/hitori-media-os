# Campaign Plan

## Campaign Thesis

記事を増やすこと自体より、AIが再利用できる形で知識を保存することが、これからの個人メディア運営の基盤になる。

## Target Reader

ひとりで複数媒体を運営し、同じアイデアをnote、X、Substack、YouTube、Podcastなどへ展開したい人。

## Positioning

Hitori Media OSは、AIで雑に量産するツールではなく、人間の主張を構造化し、媒体ごとの初稿・素材・公開準備を整えるlocal-first Media OSです。

## Core Topics

- AIが使えるDB
- 1つのContent Ideaから複数出力
- manual-first / no-API workflow
- Sanity as structured knowledge backend
- Substack as reader-list strategy layer
- video/audio/visual planning

## CTA

このテーマの続きや制作過程を追いたい人に、Substack購読またはnote/Xでのフォローを促します。

## Platform Map

- note: 論考として読ませる
- Substack: 制作ログと信頼形成
- X: フックと図解で発見を作る
- Threads: 会話調に分解する
- Instagram: 図解カルーセルで保存される
- YouTube: 体系的に説明する
- Shorts: 短いhookで入口を作る
- Podcast: 作り手の思考を音声で届ける

## Publishing Order

1. X / Threads teaser
2. note article
3. Substack Post
4. Instagram carousel
5. YouTube long-form outline or recording
6. Shorts cutdown
7. Podcast commentary
8. Substack Notes follow-up

## Reuse Map

- note article -> Substack Post
- note article -> X thread
- note article -> Threads discussion
- note article -> Instagram carousel
- note article -> YouTube script
- YouTube script -> Shorts
- YouTube / Podcast notes -> Substack Notes

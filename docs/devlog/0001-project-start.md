# 0001: プロジェクト開始

日付: 2026-05-11

## 背景

Sanity AI Content OS は、クリエイター向けのコンテンツ運用OSとしてリポジトリの土台づくりから始めます。

このプロジェクトでは、汎用ブログテンプレートや API 先行の自動化プロジェクトにならないことが重要です。

## 決定・変更

最初のリポジトリには、コンセプト、ロードマップ、スキーマ設計、API なしワークフロー、プロンプトテンプレート、ローカル入出力フォルダ、devlog 構造を入れました。

この段階では、Next.js アプリ、Sanity Studio、有料 AI API 連携は含めません。

## 理由

自動化する前に、まず手動でワークフローを証明する必要があります。

このプロダクトの主なリスクは、AI API が文章を生成できるかどうかではありません。構造化された知識レコードが、複数プラットフォームで使える質の高い出力を支えられるかどうかです。

## 影響

このリポジトリは、次のための土台になりました。

- プロダクトコンセプトの定義
- スキーマ設計の改善
- プロンプトワークフローの検証
- ローカル生成物の保存
- 判断履歴の蓄積

## 次の一手

サンプルのコンテンツアイデアから最初の出力を作り、うまくいく点・壊れる点を見ながらプロンプトとスキーマを改善します。

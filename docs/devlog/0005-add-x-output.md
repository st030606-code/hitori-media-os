# 0005: X出力を追加

日付: 2026-05-11

## 背景

Phase 1 の出力として、Threadsに続いてXを追加しました。

XはThreadsと近い短文SNSですが、役割は少し違います。

Threadsは短い連投で考えを広げる場所として扱い、Xはより短く鋭い単発投稿、GitHub更新、図解投稿、note / Substack への導線として扱います。

## 決定・変更

次を追加しました。

- `outputs/x/.gitkeep`
- `prompts/generate-x-post.md`

また、次のドキュメントにXを追加しました。

- `docs/01-roadmap.md`
- `docs/02-schema-design.md`

今回、実際のX投稿出力はまだ作成していません。

## なぜThreadsの後にXを追加したか

Threadsで、ひとつの構造化レコードから短い連投を作れることは確認できました。

ただし、Xではより短い単発投稿、図解とのペア、GitHub更新、note / Substack への配布導線が重要になります。

そのため、Threadsとは別のプロンプトとして扱うことにしました。

## XとThreadsの違い

Threads:

- 少し長めの連投で考えを広げる
- 1投稿1メッセージで順番に読ませる
- 発信者の考え方や文脈を伝えやすい

X:

- 単発で刺さる短い主張を作る
- GitHub更新や制作ログの告知に使いやすい
- 図解と組み合わせた投稿に向いている
- noteやSubstackへの導線として使いやすい

## Xが役立つ場面

Xは、次の用途で役立ちます。

- GitHubの更新を短く知らせる
- 図解やカルーセルの要点を添える
- note記事の中心主張を短く出す
- Substackの制作ログへ誘導する
- 短い仮説や学びをテストする

## 次にテストすること

次は、`inputs/content-ideas/example-ai-blog-db.json` と `prompts/generate-x-post.md` を使って、実際のX出力を1ファイルだけ作ります。

確認したいこと:

- Threadsより短く鋭い投稿になっているか
- `coreThesis` が保たれているか
- 図解ペア投稿が制作に使えるか
- GitHub / note / Substack へのCTAが自然か
- 大げさな断定や釣り表現になっていないか

## 最初のX出力で分かったこと

`outputs/x/2026-05-11--ai-blog-db--x.md` を作成しました。

Xでは、Threadsよりも短く、単体で読める主張にする必要がありました。

特に使いやすかったのは、次の要素です。

- `coreThesis`: 投稿全体の軸
- `claims`: 単発投稿やスレッドの材料
- `examples`: Obsidian / Sanity / JSON / 複数媒体展開の説明
- `platformAngles`: note、Substack、GitHubへの導線づくり

## Threadsとの違い

Threadsは、少し長めに文脈を積み上げる連投に向いています。

Xは、短い仮説、図解ペア、GitHub更新、note / Substack への配布導線に向いています。

今回の出力では、Xは「内容を広げる場所」というより、「主張を短く置いて入口を作る場所」として使うほうが合いそうだと分かりました。

## Xプロンプトの改善余地

現時点のプロンプトでも、単発投稿、スレッド、図解ペア、CTA案は生成できます。

ただし、次の改善余地があります。

- 文字数の目安を入れる
- 図解ペア投稿では、図解タイトルと投稿文を分ける指定をより明確にする
- CTA案では、GitHub / note / Substack のどれを主導線にするか選べるようにする
- 投稿トーンを「断定強め」「控えめ」「ビルドログ寄り」などで切り替えられるようにする

## MVP 7スキーマ詳細設計へ進めるか

X出力まで試したことで、主要なPhase 1出力の型は一通り確認できました。

ローカルJSONから複数出力を作る手動ワークフローは成立しています。

次は、Sanity Studioを初期化する前に、MVP 7スキーマの詳細設計へ進むのがよいです。

特に、`platformOutput` にはXのような短文SNS出力、`diagramPlan` には図解ペア投稿との関係、`workflow` にはどのプロンプトでどの出力を作ったかを記録できるようにする必要があります。

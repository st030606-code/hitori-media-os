# Codex Exec Prompt for x-hook-main-v1 (built-in image_gen route)

Created: 2026-05-14
Purpose: Codex の **built-in image_gen tool** を使って x-hook-main-v1 の candidate 画像を生成する非対話セッション。

`codex exec --enable image_generation --dangerously-bypass-approvals-and-sandbox --cd "$PWD"` で本ファイル内容を prompt として渡す想定。

---

## Hard Rules (must obey before generating anything)

1. **Use Codex built-in `image_gen` tool directly.** Do not write Python scripts. Do not write Node.js scripts. Do not write any shell scripts that call HTTP APIs.
2. **Do not call OpenAI Images API directly.** Do not call `https://api.openai.com/v1/images/...`. Do not `curl` any image generation endpoint.
3. **Do not use or request `OPENAI_API_KEY`.** Authentication must remain ChatGPT OAuth via `~/.codex/auth.json` (`auth_mode: "chatgpt"`).
4. **Do not edit, add, or commit any file in `assets/visuals/...`.** Final asset path is human-approved through Visual Register only.
5. **Do not edit, add, or commit any file in `patches/visual-assets/...`.** Patch JSON is generated by Visual Register only.
6. **Do not modify `seed/...`.** Do not run `seed --replace`.
7. **Do not write to Sanity from code.** Do not call `@sanity/client`. Do not run `npx sanity` commands.
8. **Do not auto-post.** Do not call X / Substack / note / Threads / YouTube / Podcast APIs.
9. **Do not commit secrets / API keys / project IDs / private/ filenames.**
10. **Do not create placeholder / fake image files.** If image_gen cannot run, stop with a clear error message.

If any of the above rules would be violated, **stop and exit with an error message describing which rule blocked the action**. Do not work around the rule.

## Image Specification

- size: 1200 x 675（16:9）
- format: PNG（透過不要）
- color space: sRGB
- content: 下記 layout + style guideline に従って生成

## Layout

- 画面中央に大きな日本語見出し『発信を頑張るより、発信が回る仕組みを作る。』を 2〜3 行で配置。
- 見出しの下に小さく『Hitori Media OS / development log』。
- 構造的なノードや矢印は使わない、または使ってもごく控えめに 1〜2 個まで。
- 重要要素は中央 70% に収める（X timeline の inline preview crop は 1.91:1 / 1:1 / 4:5 可変）。

## Style

- 背景: 白〜薄いオフホワイト。
- 主要テキスト: 濃いネイビー or チャコールグレー。
- アクセント: 控えめなウォーム色 1色のみ。
- font family: sans-serif（Noto Sans JP / Inter / IBM Plex 系）。
- 余白多め、文字とコントラスト最優先。
- 装飾密度は note-hero（campaign-hero-v1.png）よりも **1段階控えめ**。

## Avoid

- 完全自動化 / 全自動化 / 稼げる / 誰でも / AIに任せる時代 / 保証 / 完成 などの煽り文言。
- 顔写真 / 人物 / AI 生成 avatar / AI clone face。
- ロボット / 脳 / AI シンボル。
- グラデーション過多、光線、ガラスフレア、ネオン、影過多。
- ロゴ風表現 / 商標 / 既存サービス公式ロゴ風表現 / 絵文字装飾。
- 有料 PDF 教材本文の引用、figure コピー。
- 実 project ID / API トークン / subscriber メール / private/ ファイル名。

## Acceptance Criteria (Codex self-check before finishing)

候補を保存する前に以下を内的に確認:

- [ ] 中央 70% に見出しが収まり、X preview crop で文字が切れない
- [ ] note-hero と base / accent / font が一貫
- [ ] note-hero よりも装飾が控えめ
- [ ] 完成品の宣伝感がない（building-in-public トーン）
- [ ] secret / 実 project ID / private/ パス / 顔写真 / 有料 PDF 内容が映っていない
- [ ] サイズが正確に 1200 × 675 px
- [ ] 形式が PNG

すべて満たすなら inbox に保存。1 つでも欠ければ、stop し問題点を報告（fake placeholder で埋めない）。

## Required Final Actions (Codex side)

1. `assets/inbox/generated/building-hitori-media-os/x-hook-main-v1/` ディレクトリ内の既存 `v00N.png` を読み取り、次の空き番号を決定。

3. 生成画像を `assets/inbox/generated/building-hitori-media-os/x-hook-main-v1/v00N.png` に保存（next available number、上書き禁止）。
4. 何もしない事項:

   - `patches/visual-assets/...` への書き込み禁止
   - Visual Register 起動・approve 操作なし
   - Sanity への書き込みなし
   - 自動投稿なし
5. 最終出力として、保存した正確なファイルパス、ピクセルサイズ、ファイルバイト数、選んだ accent color 名 1 行（参考）を平文で返す。

## Hand-off (Codex → 人間)

Codex 終了後、人間が次を実行する想定（Codex はこれらをやらない）:

1. 候補画像が note-hero とトーン一致しているか目視確認。
2. `npm run visual:register` で Visual Register を起動。
3. Inbox Review で approve & register。
4. Sanity Studio で `visualAssetPlan.x-hook-main-v1` の `localAssetPath` / `status: saved` / `reviewNotes` を手動更新。
5. `npm run publish:package -- building-hitori-media-os` で配布。

## On Failure (Codex must follow)

`image_gen` tool が利用不可（feature flag 未有効、auth scope 不一致、ChatGPT plan 不対応など）の場合:

- **stop immediately**。
- placeholder image を作らない。
- Python / Node / curl で OpenAI Images API を代替呼び出ししない。
- 標準出力に明示的に "image_gen tool unavailable: <reason>" を書いて exit。
- 人間が ChatGPT 手動 fallback path（[prompt.md](prompt.md)）に戻る。

# Claude Code 向け指示

このリポジトリは、API なしで始めるコンテンツ運用OSの土台として扱ってください。

## プロジェクトの意図

Sanity AI Content OS は、ひとりメディア運営者がひとつの構造化された知識レコードを作り、それを複数の発信物へ展開するためのシステムです。

直近のワークフローは、人間が確認しながら進める形です。

1. 構造化されたアイデアを Sanity またはローカル JSON / Markdown に保存する。
2. Claude Code、Codex、ChatGPT、Claude アプリで保存済みプロンプトを使う。
3. 生成された下書きを `outputs/` に保存する。
4. 人間が確認・編集して公開する。
5. 実際に使って分かったことをプロンプトとスキーマへ反映する。

## Claude Code のふるまい

- ドキュメント、プロンプト、サンプル、ワークフローファイルの編集を優先してください。
- 明示的に依頼されるまで、アプリの足場を追加しないでください。
- 明示的に依頼されるまで、API 連携を追加しないでください。
- 入力レコードからコンテンツを生成する場合は、対応する出力フォルダへ保存してください。
- 重要な判断をした場合は、`docs/devlog/_template.md` をもとに devlog を追加してください。

## コンテンツ品質の基準

生成コンテンツでは、次を守ってください。

- 発信者の視点を残す
- 元レコードの主張と制約を守る
- プラットフォームごとの形式に合わせる
- 読者に合った言葉で書く
- 実用的な次の行動を示す

ありがちな AI っぽい水増し、曖昧な励まし、根拠のない主張は避けてください。

## 開発ログの発信化

Claude Code は、devlog を単なる作業記録ではなく、note / Substack / Threads に転用できる素材として整理してください。

devlog には次を含めます。

- 今日の判断
- なぜその設計にしたか
- CodexとClaude Codeの役割分担
- APIなしで済ませた理由
- 発信ネタになりそうな切り口
